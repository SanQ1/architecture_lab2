**Підхід**: Rich Domain Model  
**Обґрунтування**: Використання Rich Domain Model дозволяє інкапсулювати бізнес-правила (інваріанти) безпосередньо всередині класів моделей. Це запобігає стану, де моделі є просто контейнерами даних, а логіка розпорошена по сервісах. Це робить систему більш наближеною до реального світу, де об'єкт сам відповідає за свою валідність.
